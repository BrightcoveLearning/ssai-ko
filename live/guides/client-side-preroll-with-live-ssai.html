개요

Brightcove Player를 사용하여 SSAI (서버 측 광고 삽입)로 라이브 스트림을 제공 할 때 클라이언트 측 프리 롤 광고를 삽입 할 수 있습니다. 클라이언트 측 광고의 경우이 기능은 IMA 광고를 지원합니다.

이 기능에는 다음이 필요합니다.

  • IMA3 플러그인 v3.5.0 +
  • SSAI 플러그인 v1.11.0 +
  • 브라이트코브 플레이어 버전 6.44.0+

주의 사항

  • 이 기능을 사용하는 고객은 Dynamic Delivery를 사용해야합니다.
  • IMA 및 SSAI 플러그인 만 지원됩니다 (FreeWheel 없음).
  • 현재이 기능은 페이지에서 단일 플레이어 만 지원합니다.

시작하기

실시간 SSAI 스트림으로 클라이언트 측 IMA 프리 롤 광고를 재생하려면 다음 단계를 따르세요.

  1. SSAI 지원 라이브 스트림 만들기
  2. Brightcove 플레이어 만들기
  3. Studio를 사용하여 클라이언트 측 프리롤 구현

SSAI 지원 라이브 스트림 만들기

라이브 모듈은 서버 측 광고 삽입 (SSAI)을 지원하므로 라이브 스트림 중에 서버 측 광고를 요청하고 표시 할 수 있습니다. 라이브 스트림을 만들려면 다음을 참조하세요.

  • 라이브 모듈에서 서버 측 광고 구현

Brightcove 플레이어 만들기

Video Cloud Studio를 사용하여 새 Brightcove 플레이어를 만듭니다. 자세한 내용은빠른 시작: Player문서 생성 및 스타일 지정

브라이트코브 플레이어 버전 6.44.0 이상을 사용해야 합니다.

Studio를 사용하여 클라이언트 측 프리 롤 구현

자동 장애 조치 광고용으로 플레이어를 구성하는 가장 쉬운 방법은 Video Cloud Studio를 사용하는 것입니다. 광고 구성 및 플레이어를 생성했으면 다음과 같이 자동 장애 조치를 위해 플레이어를 구성 할 준비가 된 것입니다.

  1. 열기선수모듈을 클릭하고 광고 기능을 추가할 플레이어를 찾습니다.
  2. 플레이어에 대한 링크를 선택하여 플레이어의 속성을 엽니 다.
  3. 왼쪽 탐색 메뉴에서광고를 선택합니다 .
  4. 클라이언트측 (IMA) 사용확인란을 선택합니다.

  5. IMA 광고 태그의 URL을 포함하세요. 이 예에서는 샘플 광고 태그 URL을 사용합니다.

    플레이어 광고 속성에 대한 자세한 내용은 플레이어 모듈을 사용하여 플레이어 광고구성문서를 참조하십시오.

  6. 서버 측 (SSAI) 사용확인란을 선택합니다.

    귀하의 계정이 SSAI에 대해 활성화되어 있지 않으면 서버 측 옵션이 나타나지 않습니다.
  7. 구성선택드롭다운 메뉴에서 이 플레이어와 연결하려는 광고 구성을 선택합니다.

  8. 광고 위에 오버레이를 표시하려면 광고 정보 오버레이사용확인란을 선택합니다. 여기에는 '자세히 알아보기'및 광고 카운트 다운 오버레이가 포함됩니다.

    오버레이는 여기에서 활성화하고 VAST 광고 태그에서 구성할 때까지 표시되지 않습니다.
  9. 저장버튼을 선택합니다.

  10. 왼쪽 탐색 메뉴에서 JSON 편집기를 선택합니다 .

  11. JSON 편집기에서ad_failover: true속성이 표시될 때까지 아래로 스크롤합니다.

  12. ad_failover: true속성을 다음과 같이 바꿉니다. 

    "ima_preroll_with_ssai": true

  13. JSON 편집기는 다음과 유사해야합니다.

  14. 저장버튼을 선택합니다.

  15. 플레이어를 게시하려면 [ 게시 및 포함] > [변경 사항 게시] 를 선택합니다 .

  16. 이제 라이브 이벤트를 게시 할 준비가되었습니다. 자세한 내용은라이브 모듈에서 서버 측 광고 구현문서.
