소개

Brightcove의 Live SSAI 용 플레이어 향상 기능을 사용하면 광고 카운트, 광고 카운트 다운 타이머, 클릭 스루 광고 및 컴패니언 광고를 통해 라이브 스트림에서 광고 시간을 개선 할 수 있습니다.

광고 정보는 표시 할 광고를 식별하기 위해 VAST 응답을 검색하는 광고 구성을 사용하여 지정됩니다. VAST 파일에는 컴패니언 광고 및 비콘 URL에 대한 정보도 포함되어 있습니다.

현재 이 기능은 다음 리전에서만 사용할 수 있습니다.
  • ap-southeast-1 (싱가포르)
  • ap-southeast-2 (시드니)
  • ap-northeast-1 (도쿄)
  • eu-central-1 (프랑크푸르트)
  • us-west-2 (오레곤)
개략적인 개요 및 기타 제한 사항은개요를 참조하십시오. 라이브 SSAI문서에 대한 플레이어 개선 사항

워크 플로우

SSAI 및 플레이어 향상 기능을 사용하여 라이브 스트림을 만들려면 다음 단계를 따르세요.

  1. 계정 관리자에게 라이브 스트림, SSAI (Server-Side Ad Insertion) 및 Live SSAI 용 플레이어 향상을 위해 계정을 활성화하도록 요청하십시오.
  2. 광고 구성 만들기
  3. 선택 과목: 사용하지 않는 광고 시간을 채우기 위해 슬레이트 애셋 만들기
  4. 라이브 직업 만들기
  5. 재생 토큰 만들기
  6. 플레이어 구성
  7. 스트리밍 시작
  8. 큐 포인트를 사용하여 광고 트리거 (실시간 이벤트 전 또는 도중에 수행 가능)
  9. 대청소

광고 구성 만들기

광고 구성은 실시간 스트림에 서버 측 광고를 포함하는 데 필요한 정보를 정의합니다. 여기에는 광고 제공 업체 URL, 예상 광고 응답 유형 및 비콘 전송 여부가 포함됩니다.

참고 사항:
  • Video Cloud의 광고 구성 ID는 Live SSAI에 직접 사용할 수 없습니다.
  • Video Cloud Studio를 사용하여 광고 구성을 변경하면client_options플래그가 삭제됩니다. 이는 알려진 문제입니다.

새 광고 구성을 만들려면 다음과 같이POST요청을 보냅니다.

방법 POST
URL https://api.bcovlive.io/v1/ssai/applications
머리글 X-API-KEY: your API KEY
콘텐츠 유형: 응용 프로그램/json

모든 요청은 계정이 설정 될 때 제공되는 API 키를 사용하여 인증됩니다. 키가X-API-KEY헤더에 전달됩니다.

요청 본문

요청 본문 필드에 대한 자세한 내용은 Live API 참조문서를 참조하십시오.

라이브 SSAI에 대한 플레이어 향상 기능을 사용하면 강조 표시된 필드를application_ad_configuration오브젝트 내부에 설정할 수 있습니다.

샘플 요청 본문:

{
  "application_ad_configuration": {
	"ad_configuration_description": "your ad config description",
    "ad_configuration_expected_response_type": "Vast",
	"ad_configuration_headers_for_impressions": false,
	"ad_configuration_strategy": "SingleAdResponse",
	"ad_configuration_transforms": [],
	"ad_configuration_url_format": "your ad server",
    "ad_configuration_client_sdk_enabled": true,
    "client_options": {
      "show_ad_break_remaining_time": true / false,
      "show_ad_remaining_time": true / false,
      "show_number_of_remaining_ads": true / false,
      "client_only_tracking": true / false
    }
  },
  "application_description": "your application description"
}

다음은 Live SSAI를 사용한 플레이어 향상과 관련된 광고 구성 필드입니다.

들 유형 필수 기술
ad_configuration_expected_response_type 끈 예 이 기능의 경우 광고 구성에는 다음과 같은 값이 있어야 합니다. Vast
ad_configuration_client_sdk_enabled 부울 예 이 기능을 활성화하려면 이 필드에 다음 값이 있어야 합니다. true
client_options 목적 플레이어 향상과 관련된 필드 개체
client_options.
show_ad_break_remaining_time		 부울 광고 시간의 남은 표시 시간
값: true / false
client_options.
show_ad_remaining_time		 부울 광고의 남은 표시 시간
값: true / false
client_options.
show_number_of_remaining_ads		 부울 광고 시간에 남은 광고 수 표시
값: true / false
client_options.
client_only_tracking		 부울 서버 측 추적 이벤트(노출수/사분위수) 비활성화
값: true / false
기본: false

광고 노출의 정상적인 처리가 우회되는 경우true , 대신 클라이언트에 의해 해고됩니다. 이렇게하면 노출이 더 정확하고 적시에 이루어 지지만 더 이상 추적 이벤트를 볼 수 없습니다. 클라이언트 측 광고는 특정 광고 차단기에 의해 차단될 수도 있습니다.

모든client_options필드는 선택 사항입니다. 오직client_only_tracking의 기본값이 있습니다false.

재생 토큰 만들기

플레이어와 함께 재생 토큰이 필요합니다.

재생 토큰을 생성하려면 다음과 같이POST요청을 보냅니다.

방법 POST
URL https://api.bcovlive.io/v1/jobs/your live job id/playback-token
헤더 X-API-KEY: your API KEY
콘텐츠 유형: 응용 프로그램/json

요청 본문

샘플 요청 본문:

{
   "dvr":true / false,
   "application_id":"your ad config id"
}

응답

샘플 응답:

{
 "playback_token": "live.JhQabcdefghijkl000000000abcdefghi"
}

응답 본문에는 생성 된 재생 토큰이 포함됩니다. 플레이어로 Playback API 호출을 할 때 이것을 사용하십시오.

플레이어 구성

Playback API에서 라이브 스트림을 재생하는 프로세스는 일반 Video Cloud 비디오를 재생하는 방법과 약간 다릅니다. 동영상 ID (참조 ID)를 제공하는 것 외에도 사용하려는 광고 구성을 식별하는 재생 토큰을 포함해야합니다. DVR 재생 여부도 지정해야합니다.

대청소

실시간 스트림이 완료되면 다음과 같이 스트림의 아티팩트를 정리할 수 있습니다.

  1. 스트리밍 인코더 중지

  2. 라이브 작업을 취소합니다.

    라이브 작업은 작업이 생성될 때 지정된 재연결 시간에만 사용할 수 있습니다.

    라이브 작업을 취소하려면 다음과 같이PUT요청을 보냅니다.

    방법 PUT
    URL https://api.bcovlive.io/v1/jobs/your live job id/cancel
    헤더 X-API-KEY: your API KEY
    콘텐츠 유형: 응용 프로그램/json

    자세한 내용은라이브 API 참조문서를 참조하십시오.

  3. 광고 구성 삭제

    광고 구성을 삭제하려면 다음과 같이DELETE요청을 보냅니다.

    방법 PUT
    URL https://api.bcovlive.io/v1/ssai/applications/application/your ad config id
    헤더 X-API-KEY: your API KEY
    콘텐츠 유형: 응용 프로그램/json

    자세한 내용은라이브 API 참조문서를 참조하십시오.
