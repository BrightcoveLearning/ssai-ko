광고 구성은 광고 호출, 비콘 및 기타 구성 옵션을 포함하여 SSAI 재생의 다양한 측면을 정의합니다. 계정은 여러 구성을 가질 수 있으며 현재 구성은 계정간에 공유 할 수 없습니다.

Video Cloud Studio에서 광고 구성을 쉽게 생성, 나열 및 삭제할 수 있습니다. 자세한 내용은 서버 측 광고 설정구성문서를 참조하십시오.
메모:
  • SSAI를 사용하려면 Video Cloud 계정을 다음과 같이 구성해야합니다. 동적 전달 SSAI에 대해 활성화됩니다. 이 기능을 사용하려면 계정 관리자에게 문의하세요.
  • SSAI는 VOD (Video-on-Demand)와 함께 작동하며 라이브 재생을 지원합니다.
  • 제한 사항 목록은 SSAI 개요문서를 참조하십시오.

일반 정보

다음 정보는 모든 SSAI API 요청에 적용됩니다.

기본 URL

SSAI API의 기본 URL은 다음과 같습니다.

https://ssai.api.brightcove.com/v1

계정 경로

모든 경우에 특정 Video Cloud 계정에 대한 요청이 이루어집니다. 따라서 항상 용어accounts뒤에 계정 ID를 기본 URL에 추가합니다.

https://ssai.api.brightcove.com/v1/accounts/your account id

입증

요청에 대한 인증에는 Authorization 헤더가 필요합니다.

Authorization: Bearer your access token

access_token는 브라이트코브 OAuth 서비스에서 가져와야 하는 임시 OAuth2 액세스 토큰입니다. 클라이언트 자격 증명을 얻고 액세스 토큰을 검색하는 데 사용하는 방법에 대한 자세한 내용은브라이트코브 OAuth 개요를 참조하십시오 .

운영

클라이언트 자격 증명을 요청할 때 원하는 계정 액세스 또는 작업 유형을 지정해야합니다. 다음은 SSAI API에 대해 현재 지원되는 작업 목록입니다.

  • SSAI 데이터:

    video-cloud/ssai/read
    video-cloud/ssai/all

구성 필드 세부 정보

이 표는 요청의 본문 섹션에서 사용되는 광고 구성 필드를 정의합니다.

들 유형 기술
name 끈 사람이 읽을 수있는 이름
vmap_response_namespace 끈 레거시 Unicorn Once 네임 스페이스 형식을 사용하거나 새로운 Brightcove 네임 스페이스 형식을 사용하도록 VMAP 출력을 조정합니다.

값: uo , bc기본값: bc
description 끈 사람이 읽을 수있는 설명
ad_config.expected_ad_response 끈 응답을 구문 분석하는 데 사용할 기술입니다. [1]

값:
  • dfp_ad_rules
  • dfp_vmap
  • smart_xml
  • vast_3_0
ad_config.disable_server_beacons 부울 광고 노출/비콘의 서버 측 실행을 비활성화할지 여부를 플래그합니다.

로true설정하면 SSAI는 서버 측 비콘을 실행하지 않으며 모든 비콘을 VMAP 출력에 포함시킵니다.

로false설정하면 SSAI는 서버 측에서 수행할 수 있는 비콘을 발생시키고 VMAP 출력에 사용할 수 없는 비콘을 포함합니다.
ad_config.round_up_cue_points 부울 광고 삽입을 위해 가까운 위치를 선택할 때 다음 키 프레임으로 반올림할지 여부를 표시합니다.

기본값: false - 원하는 광고 위치에 가장 가까운 키프레임을 선택합니다.
ad_config.template_url.template 끈 광고 태그 템플릿. 이후 섹션에서 설명하는 사용 가능한 변수.
ad_config.template_url.defaults 목적 url 매개 변수를 제공하지 않는 경우에 사용할 기본값을 정의하는 문자열 대 문자열 맵.

리터럴이 될 수 있음{ "url.foo": "SomeString" }

또는 다른 변수 참조{% raw %}{ "url.foo": "{{ metadata.title_id }}" }{% endraw %}
discontinuities.dash [2] []String 다중 기간 대시 매니페스트를 제공 할 대시의 버전을 제어합니다.

모든 버전에 대해 여러 기간 대시를["*"]전달하려면 로 설정합니다.

[1] 때expected_ad_response로 설정됩니다vast_3_0 , SSAI는 Video Cloud에 정의된 각 광고 큐 포인트에 대해 하나의 VAST 호출을 수행합니다. VMAP 및 광고 규칙의 경우 SSAI는 초기 광고 응답에서 정의 된 광고 시간을 기반으로 요청을합니다.

[2] 불연속성 태그는 콘텐츠 비디오와 광고 세그먼트 간의 프레임 속도 인코딩 변경을 나타냅니다. 그들은 시스템에 렌더링 체인을 재구성하도록 지시합니다. 확장 가능한 SSAI의 경우 비디오 플레이어는 불연속성 태그 및 여러 기간이 포함 된 MPEG DASH를 사용하여 HLS를 재생할 수 있어야합니다.

광고 변수

템플릿 URL의 변수는 변수 경로 앞뒤에 선택적 공백을 사용하여 이중 중괄호 ( {% raw %}{{ … }}{% endraw %} ) 로 식별됩니다. 모든 변수는 다음 세 개의 네임 스페이스 중 하나가 접두어로 붙습니다.

  • 체계
  • url

시스템 변수

시스템 변수는 SSAI 시스템에서 제공하며 최종 사용자 또는 임의 값을 생성하는 도우미 변수에 대한 정보가 될 수 있습니다. 값은 URL 템플릿에 삽입하기 전에 URI로 인코딩되어야합니다.

시스템 변수는 다음과 같이 식별됩니다. {% raw %}{{system.*}}{% endraw %}

들 기술
ip_address 최종 사용자의 IP 주소
random_number_32 임의의 32 비트 정수
random_guid 무작위 UUID
referer 최종 사용자의 Referer 헤더 값
timestamp_utc 유닉스 타임 스탬프로서의 현재 시간
unique_user_id MD5 (ip_address + user_agent)
unix_timestamp 현재 시간 (Unix 타임 스탬프) (초)
user_agent 최종 사용자의 User-Agent 헤더 값
uuid 무작위 UUID
x_forwarded_for 최종 사용자의 X-Forwarded-For 헤더 값
xfp.correlator 임의 64 비트 정수
xfp.ip_address 최종 사용자의 IP 주소
xfp.unique_user_id MD5 (ip_address + user_agent)
xfp.scor 임의 64 비트 정수

진입 점 URL 매개 변수

일부 동작을 조정하기 위해 SSAI 진입 점 URL (VMAP 또는 매니페스트)에 추가 할 수있는 몇 가지 쿼리 매개 변수가 있습니다.

매개 변수 기술
?rule=sd-only 높이가 계정 구성에 설정된 임계 값보다 작은 모든 비디오 변환을 필터링합니다.
?rule=discos-enabled Dash의 HLS 및 MultiPeriods에서 불연속 재생을 활성화합니다. Playback Config의 불연속성 설정보다 우선합니다.
?rule=discos-disabled Dash의 HLS 및 MultiPeriods에서 불연속 재생을 비활성화합니다. Playback Config의 불연속성 설정보다 우선합니다.

구성 참고 사항

  1. SSAI로 광고를 미리로드해서는 안됩니다. 그 이유는 미리 로드하면 동영상이 재생되기 전에 플레이어가 광고 노출과 아마도 첫 번째 사분위수 비콘을 보고하기 때문입니다. 이는 부정확 한 광고 분석으로 이어질 수 있습니다. Studio에서 SSAI를 구성하면 자동으로 수행되지만 SSAI를 수동으로 설정하는 경우이 문제를 알고 있어야합니다.
  2. 웹 플레이어가 SSAI를 사용하고 있고 그렇게하는 동기 중 하나가 광고 차단기를 해결하는 것이라면 서버 측 비콘을 사용해야합니다. 클라이언트 측 비콘은 차단되므로 사용해서는 안됩니다.

클라이언트 측 매크로

클라이언트측 광고 매크로를 사용하려면page접두사를 사용합니다. 이러한 매크로를 사용하면 VMAP 및 서버 URL에서 변수를 사용할 수 있습니다. 광고 매크로에 대한 자세한 내용은광고 매크로 및 serverURL IMA3 플러그인을 사용한 광고 문서 섹션.

클라이언트 측 매크로는 다음으로 시작됩니다. {% raw %}{{page.*}}{% endraw %}

예를 들어, document.referrer및pageVariable DOM 창 변수를 추가하려면 접두사를 사용하여page광고 템플릿에서 다음과 같이 하십시오.

{% raw %}https://adserver.com/{{page.document.referrer}}/{{page.pageVariable.whateverIwant}}{% endraw %}

재생 API는 VMAP document.referrer및 SRC URL을 반환하고pageVariable.whateverIwant추가합니다. 그런 다음 Brightcove 플레이어는 요청을 보내기 전에 클라이언트 측 매크로 교체 로직을 실행하여 적절한 값을 교체합니다.

{% raw %}https://bolt-prefix/blah.vmap?document.referrer={document.referrer}&pageVariable.whateverIwant={pageVariable.whateverIwant}{% endraw %}

광고 오류 비콘

SSAI 사용시 VAST 광고 오류 비콘은 광고 워크 플로의 문제를 사전에 찾고 수정하는 데 도움이 될 수 있습니다. 자세한 내용은 SSAI가 포함된광고 오류 비콘문서를 참조하십시오.
