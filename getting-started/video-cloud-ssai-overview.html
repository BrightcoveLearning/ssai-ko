    비디오 클라우드 SSAI의 개요

    이 항목에서는 다이내믹 딜리버리를 위해 수집된 비디오와 함께 브라이트코브 SSAI를 사용하는 방법에 대해 알아봅니다.

    개요

    비디오를 재생할 수있는 장치의 수는 매일 확장됩니다. 특히 OTT 연결 장치는 빠른 속도로 기존 TV를 계속 대체합니다. 이러한 환경에서는 모든 디바이스 플랫폼과 모든 광고 서버 또는 네트워크에 대한 클라이언트 플레이어 통합을 구축하는 데 많은 비용과 시간이 소요될 수 있습니다. 때로는 특정 플랫폼에 대한 클라이언트 플레이어를 구축하는 것이 실용적이지 않습니다.

    브라이트코브 SSAI는 모든 디바이스에 대해 비용이 많이 드는 통합을 구축하고 유지하지 않고도 이러한 디바이스에 광고 지원 비디오를 손쉽게 전달할 수 있는 방법을 제공합니다. 브라이트코브 SSAI는 광고를 비디오 콘텐츠 스트림에 바로 연결하므로 스트리밍 비디오를 재생할 수 있는 모든 디바이스에서 작동하며, 세션별 광고 타겟팅, 클라이언트측 비코닝, 가시성 보고, 인기 광고 서비스와의 호환성이 모두 업계 최고의 Video Cloud 온라인 비디오 내에서 가능합니다.플랫폼.

    참고:

    • SSAI를 사용하려면 비디오 클라우드 계정을동적 전송으로구성하고 SSAI에 대해 활성화해야 합니다. 이 기능을 사용하려면 계정 관리자에게 문의하십시오.
    • SSAI는 VOD (주문형 비디오) 와 함께 작동하며 라이브 재생을 지원합니다.
    • 제한 사항 목록은다음 섹션을참조하십시오.

    프로세스 흐름

    SSAI가 없으면 브라우저에 광고 차단기가 있는 경우 VAST, VMAP 및 VPY를 사용하는 IMA 또는 FreeWheel 등의 클라이언트 측 광고가 재생되지 않습니다.

    SSAI와 자동 장애 조치 (Failover) 를 사용하면 광고 블록 감지가 활성화되어 광고가 항상 재생됩니다.

    • 광고 차단기가 없으면 클라이언트 측 광고가 재생됩니다.

    • 광고 차단기가 감지되면 서버 측 광고가 재생됩니다.

    SSAI에 대해 자동 장애 조치를 활성화하지 않으면 서버 측 광고만 재생됩니다.

    SSAI를 선택해야 하는 이유

    SSAI의 이점은 다음과 같습니다.

    • 부드럽고 TV와 같은 사용자 환경 - 클라이언트측 광고의 스피너, 오류 및 정지 없이 참여도를 높일 수 있습니다. 그냥 통해 재생 - 일반 TV에서와 같이!
    • 멀티 스크린 도달 - 광고 지원 VOD를 통해 OTT 연결 장치에 연결하고 싶습니까? 비디오를 가장 빠르게 성장하고 있는 비디오 시청 부문으로 푸시하고 쉽게 수익을 창출할 수 있습니다. SSAI는 비디오를 재생할 수 있는 모든 장치에서 작동, 브라이트코브 플레이어 옵션!
    • 광고 차단 손실 회수 - 광고 차단기에 손실된 수익을 즉시 10 -30% 회복합니다 . SSAI는 클라이언트 측 광고처럼 차단할 수 없습니다.
    • 세션별 타겟팅 - 동영상 메타데이터 태그, 디바이스 정보, 사용자 ID 또는 애플리케이션별 매개 변수를 각 광고 호출에 전달하여 개별 광고 타겟팅을 통해 재고에 대한 최고 CPM 비율을 얻을 수 있습니다.

    브라이트코브 플레이어와 함께 SSAI는 다음과 같은 추가 기능을 제공합니다 (VOD 전용).

    • 클라이언트측 비콘 - 타사 분석 라이브러리와의 통합을 포함하여 클라이언트의 노출, 참여도 및 조회 가능성을 보고합니다.
    • 광고 검색 사용안 함 - 광고 휴식 시간 동안 빨리 감기 기능을 비활성화합니다.
    • 광고 클릭스루 - 사용자가 클릭스루를 통해 광고 중에 자세히 알아볼 수 있습니다.
    • 컴패니언 오버레이 - 선형 동영상 광고는 컴패니언 오버레이 이미지를 가질 수

    클라이언트 플랫폼

    브라이트코브 SSAI는 광고 지원 비디오를 OTT 디바이스에 손쉽게 전달할 수 있는 방법을 제공합니다.

    웹 및 동적 전송 API용 브라이트코브 플레이어는 다양한 맞춤형 OTT 통합에도 사용할 수 있지만이러한 통합은 브라이트코브 글로벌 서비스 (BGS) 와의 컨설팅 계약을 통한 경우를 제외하고는 지원되지 않습니다 .

    API 상호 작용 또는 웹 플레이어 기능 질문은 장치별 통합 논리를 제외하는 간단한 예에서 문제를 입증할 수 있는 경우 해결됩니다.

    세션 기반 타깃팅

    광고는 다양한 플랫폼, 사이트, 앱을 대상으로 하거나 재생 시간에 알려진 다른 기준에 따라 타겟팅할 수 있으며 개별 재생 세션이 발생할 때 평가됩니다. 별도의 광고 태그를 사용하여 서로 다른 광고 재생 환경을 분할하거나 광고 매크로에 단일 광고 태그를 사용할 수 있습니다.

    매크로는 광고 요청 시 평가되므로 비디오가 재생될 때마다 클라이언트에서 전달된 변수를 참조하거나 비디오 메타데이터 또는 HTTP 요청 헤더 정보를 기반으로 SSAI 서비스에서 삽입할 수 있습니다. 매크로가 브라이트코브 SSAI에서 작동하는 방법에 대한 자세한 내용은SSAI의 구성 안내서를참조하십시오.

    SSAI는 웹에서 광고 차단기를 피할 수 있는 좋은 방법입니다. 브라이트코브 플레이어에서 SSAI를 사용하고 싶지만 VPAID 크리에이티브가 차단되지 않은 경우에도 클라이언트 측 기술을 계속 사용하려면자동 애드블록 감지 및 페일오버 기능을활성화해 보십시오.

    가시성

    트랜스코딩

    광고를 볼 수 있게 되기 전에 먼저 트랜스코딩해야 합니다. 그런 다음 스트림에 스티치 할 수 있습니다. 이는 세션 기반 타겟팅에도 적용됩니다.

    예를 들어, 하나의 광고 구성에 대해 A & B라는 두 개의 광고가있는 경우:

    • 사용자 1이 광고 A: 트랜스코딩이 시작됩니다. 사용자 1에서는 광고를 사용할 수 없습니다.
    • 사용자 2가 광고 A를받습니다 (몇 분 후). 광고 A에 대한 코드 변환이 완료되었으므로 광고
    • 사용자 1이 광고 B를 받습니다. 트랜스코딩이 시작됩니다. 사용자 1에서는 광고를 사용할 수 없습니다.
    • 사용자 3이 광고 A: 광고 A에 대한 코드 변환이 완료되었으므로 광고
    • 사용자 4는 광고 B를 얻습니다. 광고 B에 대한 코드 변환이 완료되었으므로 광고

    광고는 콘텐츠에 종속되지 않습니다. 이러한 변환은 변환에 따라 다릅니다. 따라서 위의 예에서 사용자 2가 사용자 1이 시청한 것과 동일한 변환이 설정된 콘텐츠를 시청하는 경우 광고 A를 받게 됩니다. 즉, 새 콘텐츠가 업로드되므로 일반적으로 광고를 다시 트랜스코딩할 필요가 없습니다.

    타사 메트릭

    타사 메트릭 및 가시성 시스템은 클라이언트 측 또는 서버 측 비콘을 사용하여 브라이트코브 SSAI와 함께 작동합니다. 웹용 브라이트코브 플레이어 및 SDK 플레이어는 클라이언트 측 플러그인이 직접 보고하는 데 사용할 수 있는 클라이언트 측 이벤트를 내보냅니다. 여러 공급업체에서 이 데이터를 수신하기 위해 조회 기능 및 지표 플러그인을 업데이트했습니다. 자세한 내용은 계정 관리자에게 문의하십시오.

    일반적인 클라이언트 측 광고 이벤트에 대해 자세히 알아보십시오.

    스트리밍 중 여러 지점에서 서버에서 이벤트를 내보내도록 SSAI를 구성할 수도 있습니다.

    구성 필드 세부 정보 설명서의 신호 템플릿을 참조하십시오.

    전제 조건

    SSAI는 브라이트코브 동적 딜리버리를 요구합니다 즉, 다음을 수행해야 합니다.

    1. 동적 전송 및 SSAI에 대한 계정을 활성화하려면 계정 관리자에게 문의하십시오.
    2. 동적 전송 인제스트 프로필을 사용하여 하나 이상의 새 비디오를 수집하거나 이전 비디오를 다시 트랜스코딩합니다. 자세한 내용은동적 배달 문서의 개요: Dynamic Ingest API를 참조하십시오.

    제한 사항

    SSAI 플러그인을 사용할 때는 다음 제한 사항에 유의하십시오.

    • SSAI플러그인은 브라이트코브 플레이어 v6.8.0 이상에서만 호환됩니다. v5에 대한 지원은 제공되지 않습니다.
    • SSAI를 사용하려면 콘텐츠 비디오에 오디오가 있어야 하지만다중 오디오 트랙은지원하지 않습니다.
    • 광고가 시스템에 처음 표시되는 경우 트랜스코딩되어 제공 준비가 될 때까지 재생되지 않습니다.
    • SSAI를 사용하여 공유 비디오를 재생할 때 SSAI는 하위 비디오 대신 상위 비디오의 메타데이터를 사용합니다. 하위 동영상에 로 레이블이 지정되어 있더라도 상위 동영상이 로Advertising='Free'표시된 경우 SSAI는 광고 조회를 건너뜁니다Ad Supported .

    시작하기

    동적 전송용으로 수집된 비디오와 함께 SSAI를 사용하려면 다음 단계를 수행하십시오.

    1. 광고 제공업체 또는 광고 운영팀으로부터 광고 태그를 받을 수 있습니다. 이 태그와 관련된 광고 수요가 VPLADD 광고 단위를 반환하지 않도록 구성되어 있는지 확인합니다.

    2. 웹 브라우저를 사용하여 주소 표시줄에 붙여 넣어 광고 태그를 요청합니다. 대체 매개 변수가 있는 경우 가짜 값으로 요청을 시도하거나 테스트를 위해 변수 이름을 그대로 둘 수 있습니다.

      • 오류 응답이 표시되면 먼저 광고 태그를 검토하고 올바른지 확인합니다. 매크로/변수에 대해 가짜 값을 시도해야 할 수도 있습니다.
      • 응답 유형을 살펴보십시오. 각 포드 내의 광고에 대한 VAST 문서로 연결되는 모든 광고 나누기를 보여주는 DFP 광고 규칙 응답인지 확인하십시오. 응답은VAST또는VMAP XML의 처음 몇 줄에 있습니다. DFP의 다른 XML은 아마도광고 규칙 XML이라고부르는 것입니다.
      • FreeWheel 을 사용하지 않는 경우 브라우저의 주소 표시줄에 값을 복사하여 붙여넣어 리디렉션을 수동으로 따라갈 수 있습니다. 호환되는 비디오 소스를 가리키는MediaFiles요소를 찾습니다. MP4 또는 HLS (.m3u8) 비디오 소스 파일을 찾고 있습니다.
      • 광고 응답이 로 끝나는 파일을 가리키면 VPYD 광고 단위이며 브라이트코브 SSAI에서는.js작동하지 않습니다.
    3. 광고 태그의 매개 변수를 SSAI서버로 대체할 수 있는 매개 변수와 플레이어 또는 장치에서 전달해야하는 매개 변수를 결정합니다. 광고 태그를 적절하게 작성하십시오. 광고 태그 매크로 및 해당 구문에 대한 자세한 내용은SSAI 광고 구성문서를 참조하십시오.

    4. 클라이언트 플레이어 또는 서버에서 광고에 대한 노출 및 참여 비콘을 방출할지 결정합니다.

      • 플레이어가 브라이트코브 플레이어를 사용할 수 없는 장치 (웹 또는 SDK) 에서만 사용되거나 광고 차단이 활성화된 웹 사례를 대상으로 하는 경우 서버 측 비콘을 사용하려고 합니다.
      • 일부 플랫폼에서는 서버에서 비콘을 방출하고 다른 플랫폼에서는 클라이언트에서 비콘을 방출하려면 여러 SSAI의 광고 구성을 만들어야 합니다.
         

      SSAI 광고 구성은 서버 측 비콘을 활성화하거나 비활성화할 수 있습니다. 브라이트코브 플레이어 (웹 또는 SDK) 를 사용하는 경우 클라이언트에서 이를 설정해야 합니다.

      비콘을 두 번 방출하지 않도록하십시오.
    5. 광고 태그와 비콘 설정을 사용하여 SSAI의 광고 구성 (광고 구성) 을 만듭니다. 자세한 내용은서버측 광고 설정 구성문서를 참조하십시오.
    6. 타겟팅하는 플랫폼에 관계없이 먼저 웹 플레이어를 테스트할 수 있습니다. 자세한 내용은브라이트코브 플레이어로 서버 측 광고 구현문서를 참조하십시오.
    7. 플레이어가 작동하는지 확인한 후에는위의 플랫폼 표를 기반으로 적절한 문서를 참조하여 다른 플랫폼을타겟팅할 수 있습니다.
