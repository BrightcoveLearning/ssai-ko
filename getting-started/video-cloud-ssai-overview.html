개요

비디오를 재생할 수있는 장치의 수는 매일 증가합니다. 특히 OTT 연결 장치는 기존 TV를 빠른 속도로 계속해서 대체하고 있습니다. 이 환경에서는 모든 장치 플랫폼과 모든 광고 서버 또는 네트워크에 대한 클라이언트 플레이어 통합을 구축하는 데 많은 비용과 시간이 소요될 수 있습니다. 때로는 특정 플랫폼 용 클라이언트 플레이어를 구축하는 것이 실용적이지 않습니다.

Brightcove SSAI는 모든 장치에 대해 비용이 많이 드는 통합을 구축하고 유지하지 않고도 이러한 장치에 광고 지원 비디오를 제공하는 쉬운 방법을 제공합니다. Brightcove SSAI는 광고를 비디오 콘텐츠 스트림에 바로 연결하므로 세션 별 광고 타겟팅, 클라이언트 측 비 커닝, 조회 가능성보고 및 인기있는 광고 서비스와의 호환성과 함께 스트리밍 비디오를 재생할 수있는 모든 장치에서 작동합니다. 업계 최고의 Video Cloud 온라인 비디오 플랫폼.

메모:

  • SSAI를 사용하려면 Video Cloud 계정을 다음과 같이 구성해야합니다. 동적 전달 SSAI에 대해 활성화됩니다. 이 기능을 사용하려면 계정 관리자에게 문의하세요.
  • SSAI는 VOD (Video-on-Demand)와 함께 작동하며 라이브 재생을 지원합니다.
  • 제한 사항 목록은다음 섹션을 참조하십시오 .

프로세스 흐름

SSAI가 없으면 VAST, VMAP 및 VPAID를 사용하는 IMA 또는 FreeWheel과 같은 클라이언트 측 광고는 브라우저에 광고 차단기가있는 경우 재생되지 않습니다.

SSAI 및 광고 차단 감지가 활성화 된 자동 장애 조치를 사용하면 광고가 항상 재생됩니다.

  • 광고 차단기가 없으면 클라이언트 측 광고가 재생됩니다.

  • 광고 차단기가 감지되면 서버 측 광고가 재생됩니다.

SSAI에 대해 자동 장애 조치를 활성화하지 않으면 서버 측 광고 만 재생됩니다.

왜 SSAI인가?

SSAI의 이점은 다음과 같습니다.

  • 부드럽고 TV와 같은 사용자 경험 -클라이언트 측 광고의 회 전자, 오류 및 중단없이 더 높은 참여도를 얻으십시오. 일반 TV처럼 재생됩니다!
  • 멀티 스크린 도달 -광고 지원 VOD로 OTT 연결 장치에 접근하고 싶습니까? 비디오를 가장 빠르게 성장하는 비디오보기 세그먼트로 밀어 넣고 쉽게 수익을 창출하십시오. SSAI는 비디오를 재생할 수있는 모든 장치에서 작동하며 Brightcove Player는 선택 사항입니다!
  • 광고 차단 손실 복구 -광고 차단기에 손실 된 수익 10 ~ 30 %를 즉시 되 찾으세요. SSAI는 클라이언트 측 광고처럼 차단할 수 없습니다.
  • 세션 별 타겟팅 -각 광고 호출에 동영상 메타 데이터 태그, 기기 정보, 사용자 ID 또는 애플리케이션 별 매개 변수를 전달하여 개별 광고 타겟팅을 사용하여 인벤토리에 대한 최고 CPM 요율을 얻을 수 있습니다.

Brightcove Player가 포함 된 SSAI는 다음과 같은 추가 기능을 제공합니다 (VOD 전용).

  • 클라이언트 측 비콘 - 타사 분석 라이브러리와의 통합을 포함하여 클라이언트의 노출, 참여 및 보기 가능성을 보고합니다.
  • 광고 검색 비활성화 -광고 시간에 빨리 감기를 사용 중지합니다.
  • 광고 클릭률 -사용자는 광고를 보는 동안 클릭을 통해 자세히 알아볼 수 있습니다.
  • 컴패니언 오버레이 -선형 동영상 광고는 컴패니언 오버레이 이미지를 포함 할 수 있습니다.

클라이언트 플랫폼

Brightcove SSAI는 광고 지원 비디오를 OTT 장치에 쉽게 전달할 수있는 방법을 제공합니다.

웹 및 다이내믹 딜리버리 API용 브라이트코브 플레이어는 많은 맞춤형 OTT 통합에도 사용할 수 있지만, 이러한 통합은 브라이트코브 글로벌 서비스 (BGS) 와의 컨설팅 계약을 통한경우를 제외하고는 지원되지 않습니다.

디바이스별 통합 로직을 제외한 간단한 예제에서 문제를 시연할 수 있는 경우 API 상호 작용 또는 웹 플레이어 기능 질문이 해결됩니다.

세션 기반 타겟팅

광고는 다양한 플랫폼, 사이트, 앱을 타겟팅하거나 재생시 알려진 기타 기준에 따라 개별 재생 세션이 발생할 때 평가 될 수 있습니다. 별도의 광고 태그를 사용하여 다양한 광고 재생 환경을 분할하거나 광고 매크로와 함께 단일 광고 태그를 사용할 수 있습니다.

매크로는 광고 요청시 평가되므로 동영상이 재생 될 때마다-클라이언트에서 전달 된 변수를 참조하거나 동영상 메타 데이터 또는 HTTP 요청 헤더 정보를 기반으로 SSAI 서비스에 의해 삽입 될 수 있습니다. 매크로가 브라이트코브 SSAI와 함께 작동하는 방식에 대한 자세한 내용은 SSAI 구성 가이드를 참조하십시오 .

SSAI는 웹에서 광고 차단기를 피하는 좋은 방법입니다. 브라이트코브 플레이어에서 SSAI를 사용하고 싶지만 VPAID 크리에이티브와 같은 클라이언트 측 기술을 차단하지 않을 때 계속 사용하려면자동 애드블록 감지 및 장애 조치 기능을 활성화해 보십시오 .

조회 가능성

트랜스 코딩

광고를 볼 수 있으려면 먼저 트랜스 코딩해야합니다. 그런 다음 스트림에 꿰맬 수 있습니다. 이는 세션 기반 타겟팅에도 적용됩니다.

예를 들어, 하나의 광고 구성에 대해 두 개의 광고인 A와 B가 있는 경우

  • 사용자 1이 광고 A를받습니다. 트랜스 코딩이 시작됩니다. 사용자 1은 광고를 사용할 수 없습니다.
  • 사용자 2가 광고 A를받습니다 (몇 분 후). 광고 A에 대한 트랜스 코딩이 완료되었으므로 광고
  • 사용자 1이 광고 B를받습니다. 트랜스 코딩이 시작됩니다. 사용자 1은 광고를 사용할 수 없습니다.
  • 사용자 3은 광고 A를 얻습니다. 광고 A에 대한 트랜스 코딩이 완료되었으므로 광고
  • 사용자 4는 광고 B를받습니다. 광고 B에 대한 트랜스 코딩이 완료되었으므로 광고

광고는 콘텐츠에 종속되지 않습니다. 그들은 변환에 따라 다릅니다. 따라서 위의 예에서 사용자 2가 사용자 1이 시청 한 것과 동일한 변환이 설정된 콘텐츠를 시청하는 경우 광고 A를 받게됩니다. 즉, 새 콘텐츠가 업로드되면 일반적으로 광고를 다시 트랜스 코딩 할 필요가 없습니다.

타사 메트릭

타사 메트릭 및 보기 가능성 시스템은 클라이언트 측 또는 서버측 비콘을 사용하여 브라이트코브 SSAI와 함께 작동합니다. 웹용 Brightcove Player 및 SDK 플레이어는 클라이언트 측 플러그인에서 직접보고하는 데 사용할 수있는 클라이언트 측 이벤트를 내 보냅니다. 여러 공급 업체가이 데이터를 수신하기 위해 조회 가능성 및 측정 항목 플러그인을 업데이트했습니다. 자세한 내용은 계정 관리자에게 문의하세요.

일반적인 클라이언트 측 광고 이벤트에 대해 자세히 알아보세요.

스트리밍하는 동안 다양한 지점에서 서버에서 이벤트를 내보내도록 SSAI를 구성 할 수도 있습니다.

구성 필드 세부 정보 문서에서 비콘 템플릿을 참조하십시오.

전제 조건

SSAI에는 Brightcove Dynamic Delivery가 필요합니다. 이는 다음을 수행해야 함을 의미합니다.

  1. 계정 관리자에게 문의하여 Dynamic Delivery 및 SSAI에 대해 계정을 활성화하십시오.
  2. Dynamic Delivery 수집 프로필을 사용하여 하나 이상의 새 비디오를 수집하거나 이전 비디오를 다시 트랜스 코딩합니다. 자세한 내용은개요를 참조하십시오. 동적 전송을위한 동적 인제스트 API문서입니다.

한계

SSAI 플러그인을 사용할 때 다음 제한 사항에 유의하십시오.

  • SSAI 플러그인은 Brightcove Player v6.8.0 이상과 만 호환됩니다. v5에 대한 지원은 제공되지 않습니다.
  • SSAI는 콘텐츠 비디오에 오디오가 있어야 하지만다중 오디오 트랙을 지원하지 않습니다 .
  • Google 시스템에서 광고를 처음 보게되면 트랜스 코딩되고 게재 할 준비가 될 때까지 재생되지 않습니다.
  • SSAI로 공유 된 비디오를 재생할 때 SSAI 매크로 대체는 자식 비디오 대신 부모 비디오의 메타 데이터를 사용합니다. 하위 동영상의 레이블이 로 표시되어 있어도 SSAI는 상위 동영상이 로Advertising='Free'표시된 경우 광고 조회를 건너뜁니다Ad Supported .

시작하기

Dynamic Delivery를 위해 수집 된 비디오에 SSAI 사용을 시작하려면 다음 단계를 따르십시오.

  1. 광고 제공 업체 또는 광고 운영팀에서 광고 태그를받습니다. 이 태그와 관련된 광고 수요가 VPAID 광고 단위를 반환하지 않도록 구성되어 있는지 확인하세요.

  2. 웹 브라우저를 사용하여 주소 표시 줄에 붙여 넣어 광고 태그를 요청합니다. 대체 매개 변수가있는 경우 가짜 값으로 요청을 시도하거나 테스트를 위해 변수 이름을 그대로 둘 수 있습니다.

    • 오류 응답을 받으면 먼저 광고 태그를 검토하고 올바른지 확인하세요. 매크로 / 변수에 대해 가짜 값을 시도해야 할 수 있습니다.
    • 응답 유형을보십시오. 각 광고 모음 내의 광고에 대한 VAST 문서로 연결되는 모든 광고 시간을 표시하는 DFP 광고 규칙 응답인지 확인합니다. 응답은 다음과 같이 말해야 합니다. VAST또는VMAP XML의 처음 몇 줄에 있습니다. DFP의 다른 XML은 아마도광고 규칙 XML이라고 부르는 것입니다 .
    • FreeWheel을 사용하지 않는 경우 값을 복사하여 브라우저의 주소 표시 줄에 붙여 넣어 수동으로 리디렉션을 따를 수 있습니다. 호환되는 비디오 소스를 가리키는MediaFiles요소를 찾습니다. MP4 또는 HLS (.m3u8) 비디오 소스 파일을 찾고 있습니다.
    • 광고 응답이 끝나는 파일을 가리키는 경우 VPAID 광고 단위이며 브라이트코브 SSAI에서는.js작동하지 않습니다.
  3. SSAI 서버로 대체 할 수있는 광고 태그의 매개 변수와 플레이어 또는 기기에서 전달해야하는 매개 변수를 결정합니다. 광고 태그를 적절하게 작성하십시오. 광고 태그 매크로 및 적절한 구문에 대한 자세한 내용은 SSAI 광고 구성문서를 참조하십시오.

  4. 클라이언트 플레이어 또는 서버에서 광고에 대한 노출 및 참여 비콘을 내보낼 지 여부를 결정합니다.

    • 플레이어가 Brightcove 플레이어 (웹 또는 SDK)를 사용할 수없는 장치에서만 사용되거나 광고 차단이 활성화 된 웹 케이스를 대상으로하는 경우 서버 측 비콘을 사용하려고합니다.
    • 일부 플랫폼에서는 서버에서 비콘을 내보내고 다른 플랫폼에서는 클라이언트에서 비콘을 내보내려면 여러 SSAI 광고 구성을 만들어야 합니다.
       

    SSAI 광고 구성은 서버 측 비콘을 활성화 또는 비활성화 할 수 있습니다. Brightcove Player (웹 또는 SDK)를 사용하는 경우 클라이언트에서도이를 설정해야합니다.

    비콘을 두 번 방출하지 않도록 합니다.
  5. 광고 태그 및 비콘 설정으로 SSAI 광고 구성 (광고 구성)을 만듭니다. 자세한 내용은 서버 측 광고 설정구성문서를 참조하십시오.
  6. 어떤 플랫폼을 타겟팅하든 관계없이 웹 플레이어를 먼저 테스트 해 보는 것이 좋습니다. 자세한 내용은 브라이트코브 플레이어로 서버 측 광고구현문서를 참조하십시오.
  7. 플레이어가 작동하는지 확인한 후에는 위의 플랫폼 표를 기반으로 적절한 문서를 참조하여 다른플랫폼을 타겟팅할 수있습니다.
